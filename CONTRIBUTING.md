# Contributing to Aathman Core

Welcome! We appreciate your interest in contributing to **Aathman Core**.  
This document outlines the contribution process to ensure a smooth and consistent
experience for everyone.



## 1. Contributor License Agreement (CLA)

Before we can accept your pull request, you **must sign a Contributor License Agreement (CLA)**.

This ensures that:

- You have the right to contribute the code.
- Your contribution can be legally used in both the open-source Aathman Core
  and the commercial Aathman Enterprise edition.
- The project remains legally safe for all users and contributors.

See **`CLA.md`** for full details:

- Individuals sign the **ICLA**  
- Companies sign the **CCLA**

Pull requests will be automatically blocked until the CLA is signed.



## 2. Code Guidelines

To maintain consistency and reliability, please follow these guidelines:

- Keep all fingerprinting and serialization logic **deterministic**.
- Follow existing file structure, naming conventions, and design patterns.
- Include tests when introducing new features or modifying existing logic.
- Document any new public APIs, modules, or behaviors.
- Use clear, meaningful commit messages.



## 3. Pull Request Process

Please follow these steps when submitting a PR:

1. **Fork** the repository.
2. **Create a feature branch** for your changes.
3. Implement your changes with appropriate tests and documentation.
4. Submit a **pull request** to the `main` branch.
5. **Sign the CLA** when prompted by our automated system.
6. Wait for:
   - Automated checks to pass  
   - Code review feedback
7. Apply any requested changes.
8. Once approved, a maintainer will merge your PR.



## 4. Reporting Issues

If you find a bug or want to request a feature:

- Open an issue in the GitHub Issues tab.
- Provide as much detail as possible.
- Include steps to reproduce when applicable.



Thank you for helping improve Aathman Core!
Your contributions help strengthen the global model-trust ecosystem.
